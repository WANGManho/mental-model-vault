# Mental Model Obsidian Vault

This vault is a practical implementation of Charlie Munger's **latticework of mental models**, Niklas Luhmann's **Zettelkasten** note-making method, and Obsidian's **bidirectional graph**. Use it to build a constantly evolving knowledge network that improves judgment and decision-making.

## Philosophy of the System
- **Store mental models** as evergreen notes that capture definitions, triggers, examples, and sources.
- **Connect ideas** relentlessly. Every note should link to related principles, cases, biases, and strategies to form a latticework.
- **Think in combinations**. Decisions rarely hinge on a single idea; treat each note as a building block for stacked reasoning.
- **Iterate like a Zettelkasten**. Capture fleeting thoughts in `00_System`, turn them into Principle/Model/Bias notes, and upgrade them into connected permanent notes in the appropriate folders.

## How to Add Notes
1. Start with a template from the `Templates` folder so every note follows a consistent structure.
2. Store fundamental ideas in `01_Principles`, core models in `02_Mental_Models`, supporting strategies in `03_Strategies`, and cognitive pitfalls in `04_Cognitive_Bias`.
3. Capture real examples in `05_Cases` and record your own applications in `06_Decisions`.
4. Update `07_MOC` pages whenever you add material so the high-level maps stay fresh.

## Linking Ideas
- Use Obsidian's `[[wikilinks]]` liberally. Every note has dedicated sections for **Related Models** or **Models Used**.
- When you link two notes, skim the target note and reciprocate links if the relationship is meaningful. Bidirectional links power the graph view.
- Tag selectively (e.g., `#mental-model`, `#principle`, `#decision`) so filters stay useful.

## Using the Graph View
- From Obsidian, open the local graph for a note to see its immediate latticework. Aim for at least three meaningful connections per note.
- Use the global graph with filters such as `tag:#mental-model` or `path:05_Cases` to surface clusters and bridge disciplines.
- Periodically review sparse nodes—add links, supporting cases, or strategies so they integrate with the network.

## Growing the Knowledge Network
- Convert reading highlights into Literature or Fleeting notes in `00_System`, then promote them into permanent notes using templates.
- Build decision logs that cite specific models; revisit them to evaluate outcomes and refine your playbook.
- Create new Map of Content (MOC) pages whenever a cluster of ideas deserves its own hub. The `Mental Models Map` is a starting point—expand it as the vault grows.

Happy thinking—keep compounding insight by linking ideas across domains.
