# Mental Model: Path Dependence

## Definition
Path dependence explains how early decisions or random events lock systems into trajectories that are difficult to reverse.

## Core Principle
Switching costs, standards, and installed base effects create inertia, so history shapes current options.

## When To Use
- Evaluating technology standards and ecosystems.
- Understanding organizational culture or incentive structures.

## Examples
- QWERTY keyboard layout persisting despite alternatives.
- [[Amazon Flywheel]] investments compounding into a moat that competitors struggle to replicate.

## Related Models
[[Network Effects]] · [[Opportunity Cost]] · [[Evolution]]

## Opposite / Failure Cases
- Blank-slate redesigns enabled by disruptive tech.
- Modular architectures that lower switching costs.

## Sources
- Economic history studies on increasing returns.
